# P8 Task 2 — Studio UX Design (controller-authored; implementation is transcription)

Harmony pass, FAI overlay. Register: the studio is a *tool* — IBM formal register (rules, mono labels,
hard edges), not the exuberant register. The banners are the color; the chrome stays quiet.

## Panel restructure

Current seven groups (Size, Template, Program, Accent, Density, Figures, Seed) follow implementation
history, not decision order. New structure — four groups, top-to-bottom = the order a user decides:

1. **Size** — the six arrangements as segmented text chips (reuse `.chips`/`.chip.on`), replacing the
   select. Labels: `Banner 6×3 · Portrait 2×3 · Square 3×3 · Strip 3×1 · Column 1×6 · Column 1×3`.
2. **Color** — the single-hue law made visible instead of a mystery disabled control:
   - Row 1: Program select (Auto default) — unchanged options.
   - Row 2: the seven accent swatches as toggle buttons (spec below).
   - No program: swatches free-toggle → `config.accentPool` = checked set; none checked = Auto (canon);
     all checked = full palette. Two text presets right-aligned in the group header: `none` / `all`.
   - Program chosen: that program's swatch renders on+locked, the other six dim to 40% with
     `cursor: not-allowed`; presets hidden. (Chips ARE the explanation of the law.)
   - Caption line (11px Plex Mono, `--soft`) states the mode: `canon mix` / `N accents` / `full palette` /
     `program hue`.
3. **Pattern** — Template select, Density slider, Figures toggle merged into one group (three texture
   knobs, one decision). Inline `.row` labels.
4. **Seed** — unchanged (input, reroll, variations).

## Accent swatch spec

- Button 28×28, border-radius 0 (hard-edged per FAI), background = the hex, 1px `--line` border;
  7 in one wrapped row, 6px gap. Orange first, then alphabetical (Celestial Blue, Chrome Yellow,
  Electric Violet, Frontier Indigo, Signal Green, Telemagenta). `title="{Name} {hex}"`.
- ON state: 2px `--ink` outline, offset 2px, PLUS a check glyph centered in the contrast-correct
  neutral for that hex (smoke-white on dark hues, cod-gray on light — reuse the engine's luminance
  rule threshold; do not eyeball per-hue). Outline must not be orange (invisible on the orange swatch).
- States (all seven, mandatory): default / hover (outline 1px `--soft`) / focus-visible (2px
  `#FF4F00` ring — the FAI focus rule; ring sits OUTSIDE the ink outline) / active (translateY 1px) /
  on / disabled (40% opacity) / on+locked (program mode: on-state + disabled cursor).
- `aria-pressed` on each; the row is `role="group" aria-label="Accents"`.

## Config & persistence

- `config.accentPool?: string[]` replaces `paletteMode` + `accent` in the UI. Engine mapping:
  0 checked → omit both knobs (auto); 1..7 checked → `accentPool` (engine handles 7 ≡ full).
- Migration on load: persisted `paletteMode:'full'` → all seven checked; persisted `accent: hex` →
  that one checked; unknown hexes dropped (validate against the locked 7).
- The explicit-accent select is REMOVED. (Single accent = one chip checked; engine path identical
  per Task 1's equivalence guarantee.)

## Chrome polish (whole panel)

- Vertical rhythm: 8px base; group gap 20px; keep the 2px `--ink` top rule + h3 (it's right).
- All labels: 12px; controls text 13px; kill any px drift. No new colors, no new greys — tokens only.
- Buttons: existing style, but unify heights (28px) and focus rings (orange) across select/chip/button.
- Save tray + canvas actions untouched this task except focus rings.
- Mobile (≤768px): panel groups stack (existing behavior), chips wrap to two rows of 4+3; verify
  320px no horizontal scroll.
